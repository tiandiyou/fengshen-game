# 抽卡系统设计文档

**版本**: v4.0  
**更新时间**: 2026-03-15

---

## 一、卡包分类

### 1.1 卡包类型

| 卡包 | 货币 | 品质 | 说明 |
|------|------|------|------|
| 大卡包 | 元宝 | 红/橙/紫 | 高品质卡池 |
| 小卡包 | 元宝 | 橙/紫/蓝 | 日常卡池 |
| 紫将卡包 | 灵气 | 紫/蓝 | 低价卡池 |

### 1.2 卡包定位

```
大卡包: 抽红将/橙将
小卡包: 抽橙将/紫将
紫将卡包: 抽紫将/蓝将 (保底)
```

---

## 二、大卡包

### 2.1 抽卡消耗

```
单抽: 280元宝
十连: 2500元宝 (省300)
```

### 2.2 概率分布

| 品质 | 概率 |
|------|------|
| 红 | 1.5% |
| 橙 | 8.5% |
| 紫 | 40% |
| 蓝 | 50% |

### 2.3 保底机制

```
20抽 → 必出橙将
累计6橙 → 必出红将
```

### 2.4 武将池

```
包含所有红将/橙将/紫将
```

---

## 三、小卡包

### 3.1 抽卡消耗

```
单抽: 100元宝
十连: 900元宝 (省100)
```

### 3.2 概率分布

| 品质 | 概率 |
|------|------|
| 橙 | 5% |
| 紫 | 35% |
| 蓝 | 60% |

### 3.3 保底机制

```
20抽 → 必出紫将
累计10紫 → 必出橙将
```

### 3.4 武将池

```
只包含橙将/紫将
无红将
```

---

## 四、紫将卡包

### 4.1 抽卡消耗

```
单抽: 100灵气
十连: 900灵气 (省100)
```

### 4.2 概率分布

| 品质 | 概率 |
|------|------|
| 紫 | 20% |
| 蓝 | 80% |

### 4.3 保底机制

```
10抽 → 必出紫将
累计20紫 → 必出紫将(保底刷新)
```

### 4.4 武将池

```
只包含紫将/蓝将
无橙将/红将
```

### 4.5 定位

```
紫将卡包 = 日常保底卡池
适合:
- 新手玩家
- 灵气多余时
- 刷紫将狗粮
```

---

## 五、名将加成

### 5.1 加成说明

```
名将效果只影响大卡包概率
不影响小卡包/紫将卡包
```

### 5.2 加成表

| 名将 | 大卡包红将概率 |
|------|----------------|
| 无 | 1.5% |
| 良将 | 2.5% |
| 名将 | 4.5% |
| 神将 | 6.5% |

## 七、数据结构

### 7.1 卡包配置

```java
class GachaPack {
    Long id;
    String name;        // 大卡包/小卡包/紫将卡包
    String type;        // big/small/purple
    
    Integer cost;       // 单抽价格
    Integer cost10;     // 十连价格
    
    // 概率
    Double redRate;
    Double orangeRate;
    Double purpleRate;
    Double blueRate;
    
    // 保底
    Integer pityCount;     // 保底抽数
    String pityQuality;   // 保底品质
    
    Integer orangePity;   // 橙将保底累计
    Integer purplePity;   // 紫将保底累计
    
    // 武将池
    List<Integer> redPartners;   // 红将ID列表
    List<Integer> orangePartners; // 橙将ID列表
    List<Integer> purplePartners; // 紫将ID列表
    List<Integer> bluePartners;  // 蓝将ID列表
}
```
