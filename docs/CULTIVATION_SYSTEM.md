## 一、核心设计

### 1.1 修仙分支

```
修仙分为两大流派:
- 体修 (物理系)
- 法修 (法术系)

玩家只能选择一种流派修炼
```

### 1.2 修仙效果

| 分支 | 效果 |
|------|------|
| 体修 | 增加武将属性 (武力/统率/速度) |
| 法修 | 增加技能强度 (伤害加成) |

### 1.3 修炼方式

```
修仙产出: 灵气结晶
↓
消耗灵气结晶进行修炼
↓
资源投入越多，修炼越快
↓
产出对应方向的加成
```

---

## 二、体修系统

### 2.1 体修等级

```
体修等级: 1-50级
```

### 2.2 升级消耗

| 等级 | 升级所需结晶 | 基础时间 |
|------|--------------|----------|
| 1→2 | 100 | 1小时 |
| 2→3 | 200 | 2小时 |
| ... | ... | ... |
| 49→50 | 4900 | 49小时 |

**免费时间**: 1小时/级  
**加速**: 投入结晶可立即完成

### 2.3 属性加成

```
每级体修:
- 武力 +1
- 统率 +0.5
- 速度 +0.5

50级体修:
- 武力 +50
- 统率 +25
- 速度 +25
```

### 2.4 加成方式

```
体修属性 = 加成到武将基础属性

示例:
哪吒武力: 160
体修50级: +50武力
最终武力: 210
```

## 三、法修系统

### 3.1 法修等级

```
法修等级: 1-50级
```

### 3.2 升级消耗

同体修

### 3.3 技能强度加成

```
每级法修:
- 物理伤害 +0.5%
- 法术伤害 +0.5%

50级法修:
- 物理伤害 +25%
- 法术伤害 +25%
```

### 3.4 加成方式

```
法修强度 = 加成到所有技能伤害

示例:
火尖枪基础伤害: +55%
法修50级: +25%伤害
最终伤害: +80%
```

## 四、资源加速

### 4.1 免费修炼

```
每级有1小时免费修炼时间
时间到达后修炼完成
```

### 4.2 加速消耗

```
加速公式: 每提前1小时完成 = 消耗100灵气结晶

示例:
1级升2级需要1小时
- 免费等待1小时
- 或消耗100结晶立即完成

49级升50级需要49小时
- 免费等待49小时
- 或消耗4900结晶立即完成
```

### 4.3 加速示例

```
升级需要: 1000结晶 (10小时)
选择加速: 5小时
消耗结晶: 5 × 100 = 500结晶
节省时间: 5小时
```

## 五、修仙选择

### 5.1 流派选择

```
创角时选择修仙流派:
- 选择体修
- 选择法修

流派一旦选择不可更改
```

### 5.2 流派特点

| 流派 | 适合 | 加成 |
|------|------|------|
| 体修 | 物理将阵容 | 武力/统率/速度 |
| 法修 | 法术将阵容 | 技能伤害 |

### 5.3 阵容推荐

```
体修阵容:
- 哪吒、杨戬、赵公明、闻仲
- 物理伤害为主

法修阵容:
- 姜子牙、妲己、申公豹、杨任
- 法术伤害为主
```

## 六、数据结构

### 6.1 修仙数据

```java
class Cultivation {
    Long id;
    Long playerId;
    
    String type;  // "body"=体修, "magic"=法修
    
    Integer level = 1;    // 修仙等级
    Integer exp = 0;      // 当前经验
    
    LocalDateTime lastUpdate; // 上次更新时间
    
    // 体修加成
    Integer getAtkBonus() { return level * 1; }
    Integer getLeadBonus() { return level * 1 / 2; }
    Integer getSpeedBonus() { return level * 1 / 2; }
    
    // 法修加成
    Double getDamageBonus() { return level * 0.5 / 100.0; }
}
```

### 6.2 升级逻辑

```java
// 升级所需结晶
int getUpgradeCost(int currentLevel) {
    return currentLevel * 100;  // 1级=100, 2级=200...
}

// 基础时间(小时)
int getBaseTime(int currentLevel) {
    return currentLevel;  // 1级=1小时
}

// 加速消耗
int getAccelerateCost(int currentLevel, int hoursToAccelerate) {
    return hoursToAccelerate * 100;
}
```

## 七、跨赛季规则

```
赛季结束:
- 修仙等级: ✅ 保留
- 修仙流派: ✅ 保留
- 修仙经验: ✅ 保留

新赛季:
- 从1级重新修炼
- 可用上赛季积累的结晶
```
