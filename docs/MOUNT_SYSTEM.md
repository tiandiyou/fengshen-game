**版本**: v1.0  
**更新**: 2026-03-15  
**定位**: SLG赛季制

---

## 一、核心设计

### 1.1 坐骑定位

```
坐骑 = 额外装备位
每个武将可携带1个坐骑
坐骑提供少量属性加成
```

### 1.2 跨赛季规则

```
跨赛季: 坐骑不保留
```

---

## 二、坐骑属性

### 2.1 属性加成

```
坐骑属性幅度类似装备

| 类型 | 加成 |
|------|------|
| 武力 | +10~30 |
| 智力 | +10~30 |
| 统率 | +10~30 |

## 四、坐骑获取

### 4.1 遗迹系统

```
大地图上有遗迹
玩家每天可攻打1次
胜利获得坐骑
```

### 4.2 遗迹类型

| 遗迹 | 品质 | 难度 |
|------|------|------|
| 山洞 | 蓝/紫 | 简单 |
| 古墓 | 紫/橙 | 中等 |
| 仙府 | 橙/红 | 困难 |

### 4.3 攻打规则

```
次数: 每天1次
时间: 重置时间0点
进入: 点击遗迹进入战斗
战斗: 与坐骑守护战斗
胜利: 获得随机坐骑
失败: 无奖励
```

### 4.4 掉落概率

```
山洞:
- 蓝色: 60%
- 紫色: 35%
- 橙色: 5%
- 红色: 0%

- 红色: 2%

仙府:

## 六、坐骑穿戴

### 6.1 穿戴规则

```
每个武将可携带1个坐骑
坐骑可自由拆卸
坐骑属性直接叠加到武将
```

### 6.2 属性计算

```
最终属性 = 武将属性 + 坐骑属性

示例:
武将武力: 100
坐骑麒麟: +30
最终武力: 130
```

## 七、数据结构

### 7.1 坐骑

```java
class Mount {
    Long id;
    String name;          // 坐骑名称
    String quality;       // 品质: red/orange/purple/blue
    
    // 属性
    Integer atk;         // 武力加成
    Integer intelligence; // 智力加成
    Integer lead;         // 统率加成
    Integer speed;        // 速度加成
    Integer crit;         // 暴击加成
    
    // 特技
    String skill;        // 特技名称
    String skillLevel;  // 特技等级: S/A/B
    
    Long playerId;
    Long partnerId;      // 装备在哪个武将身上(0=未装备)
}
```

### 7.2 遗迹记录

```java
class RuinsRecord {
    Long id;
    Long playerId;
    String ruinsType;    // 山洞/古墓/仙府
    
    LocalDateTime lastAttack; // 上次攻击时间
    Integer todayCount;     // 今日攻击次数
}
```
