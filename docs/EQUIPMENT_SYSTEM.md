# 装备系统与材料系统设计

**版本**: v3.0  
**更新**: 2026-03-15  
**定位**: SLG赛季制

---

## 一、设计原则

1. **材料来源**: 大地图行军遇怪掉落
2. **装备合成**: 4材料=1装备
3. **跨赛季**: 装备/材料不保留
4. **简化**: 去掉强化/锻造复杂功能

---

| 材料 | 品质 | 颜色 | 用途 |
|------|------|------|------|
| 玄铁 | 1星 | 灰色 | 蓝装合成 |
| 精钢 | 2星 | 蓝色 | 紫装合成 |

|----------|----------|----------|

```

- 必掉1个精钢

```

## 六、合成系统

### 6.1 合成公式

```
4个同名材料 = 1件对应品质装备

示例:
4个玄铁 → 1件蓝色武器
4个精钢 → 1件紫色防具
4个陨金 → 1件橙色鞋子
4个天晶 → 1件红色饰品
```

### 6.2 合成概率

```
成功概率:
- 蓝→紫: 100%
- 紫→橙: 80%
- 橙→红: 60%

失败:
- 返还2个同品质材料
- 不返还高阶材料
```

### 6.3 合成示例

```
合成紫色武器:
消耗: 4×精钢
结果: 紫色武器 (属性随机)
      30%概率带特技
```

## 七、装备穿戴

### 7.1 穿戴规则

```
每个武将最多穿戴4件装备:
- 武器 ×1
- 防具 ×1
- 鞋子 ×1
- 饰品 ×1

专属装备只能给对应武将穿戴
```

### 7.2 属性计算

```
最终属性 = 武将基础属性 + 装备属性

示例:
武将武力: 100
装备武器: +30攻击 (紫色)
最终武力: 130
```

## 九、跨赛季规则

```
赛季结束:
- 装备: ❌ 不保留 (消失)
- 材料: ❌ 不保留 (清空)
- 特技: ❌ 不保留

新赛季:
- 从玄铁开始打材料
- 重新合成装备
```

## 十、数据结构

### 10.1 材料

```java
class Material {
    Long id;
    String name;    // 玄铁/精钢/陨金/天晶
    Integer star;   // 1/2/3/4
    Integer count;  // 数量
    Long playerId;
}
```

### 10.2 装备

```java
class Equipment {
    Long id;
    String name;
    String type;    // weapon/armor/shoes/accessory
    String quality; // blue/purple/orange/red
    
    // 属性
    Integer atk;
    Integer def;
    Integer speed;
    Integer crit;
    
    // 特技
    String skill;   // 猛攻/破甲/疾风/闪避/命中
    String skillLevel; // S/A/B
    
    // 专属
    Integer partnerId; // 0=非专属
    
    Long playerId;
    Integer partnerIdEquipped; // 装备在哪个武将身上
}
```
