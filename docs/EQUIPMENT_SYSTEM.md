# 装备系统设计文档 (简化版)

**版本**: v2.0  
**更新时间**: 2026-03-15

---

## 一、核心设计原则

1. **无强化系统** - 装备属性固定
2. **无生命值** - 只有兵量
3. **材料合成** - 锻造=材料合成装备
4. **大地图获取** - 材料通过野怪掉落

## 六、合成系统

### 6.1 合成公式

```
4个同品质材料 = 1件对应品质装备

示例:
4个玄铁 = 1件蓝色武器
4个精钢 = 1件紫色武器
4个陨金 = 1件橙色武器
4个天晶 = 1件红色武器
```

### 6.2 合成概率

```
成功概率:
- 蓝装: 100%
- 紫装: 80% (失败返还2个材料)
- 橙装: 60% (失败返还2个材料)
- 红装: 40% (失败返还2个材料)
```

### 6.3 特技概率

```
合成时随机判定特技:
- 蓝装: 5%
- 紫装: 10%
- 橙装: 15%
- 红装: 20%
```

## 七、大地图系统

### 7.1 地图结构

```
大地图 = 多个区域组成
每个区域 = 多个关卡
每个关卡 = 若干野怪
```

### 7.2 野怪类型

| 类型 | 特点 | 材料掉落 |
|------|------|----------|
| 散兵 | 简单 | 玄铁 |
| 山贼 | 简单 | 玄铁 |
| 流寇 | 中等 | 玄铁+精钢 |
| 叛军 | 中等 | 精钢+陨金 |
| 妖道 | 困难 | 陨金 |
| 魔将 | 精英 | 陨金+天晶 |
|  Boss | 首领 | 天晶(必掉) |

### 7.3 战斗消耗

```
每次战斗消耗: 兵量 × 10%

胜利: 获得材料
失败: 获得少量材料 (50%)
```

## 八、数据结构

### 8.1 装备实体

```java
class Equipment {
    Long id;
    String name;          // 装备名
    String type;         // weapon/armor/shoes/accessory
    String quality;      // red/orange/purple/blue
    
    // 属性
    Integer atk;         // 攻击(武力/智力)
    Integer def;         // 防御
    Integer speed;       // 速度
    Integer crit;        // 暴击
    
    // 特技
    String skill;        // 特技名
    String skillLevel;   // S/A/B
    
    // 专属
    Integer partnerId;   // 专属武将ID (0=非专属)
    
    // 状态
    Boolean equipped;
    Long ownerId;
}
```

## 十、总结

### 系统简化点

1. ❌ 无强化系统 - 装备属性固定
2. ❌ 无生命值 - 只有兵量
3. ❌ 无锻造 - 改为材料合成
4. ✅ 保留特技 - 战斗少量加成
5. ✅ 保留专属 - 武将专属装备

### 核心循环

```
大地图打怪 → 获取材料 → 材料合成装备 → 装备属性提升 → 打更强的怪
```
